HOW DATA TRAVEL?

DATA Can Travel Over Internet Through Diffrent Processes Which are:

1. IP ADDRESS:

An Internet Protocol (IP) address is a unique numerical identifier for every device or network that connects to the internet. Typically assigned by an internet service provider (ISP), an IP address is an online device address used for communicating across the internet.

    TYPES OF IP ADDRESS.

  1. IPv4 (Internet Protocol version 4):

    • Format: 32-bit numeric address written as four decimal numbers separated by periods (e.g., 192.168.1.1).
    • Example: 172.16.254.1
    • Limited address space, supporting about 4.3 billion unique addresses.

  2. IPv6 (Internet Protocol version 6):

    • Format: 128-bit alphanumeric address written as eight groups of four hexadecimal digits, separated by colons (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334).
    • Example: 2001:0db8:85a3::8a2e:0370:7334
    • Expanded address space to accommodate the growing number of internet-connected devices.

2. DOMAIN NAME:

A domain name is a human-readable address used to access websites on the internet. Instead of using a numeric IP address (e.g., 192.168.1.1), domain names allow people to easily remember and find websites. A domain name typically consists of two main parts:

PARTS OF DOMAIN NAME.

  1. Second-level domain (SLD):

    2. This is the main part of the domain, like "google" in google.com.

  2. Top-level domain (TLD):

    3. Top-level domain (TLD): This is the extension that comes after the SLD, like ".com", ".org", ".net", or country-specific ones like ".ng" for Nigeria.

For example, in "www.example.com":

  1. "example" is the SLD
  2. ".com" is the TLD

Domain names are translated into IP addresses using a system called DNS (Domain Name System), allowing users to reach a website without memorizing IP addresses.


3. DOMAIN NAME SYSTEM:

The Domain Name System (DNS) is a system that translates human-readable domain names (like www.example.com) into machine-readable IP addresses (like 192.0.2.1). This process allows users to access websites and services on the internet using easy-to-remember domain names rather than needing to remember numerical IP addresses.

How DNS Works:

  • User Types a Domain Name:

    • When a user enters a domain name into their browser, the browser first checks if it has the IP address stored locally (in a cache).

  • Query to DNS Resolver:

    • Query to DNS Resolver: If not cached, the request is sent to a DNS resolver (usually provided by the internet service provider) which begins the lookup process

  • Recursive Queries:
    • The DNS resolver contacts Root DNS servers, which point it to the TLD DNS servers (e.g., for .com, .org, etc.).
    • The TLD DNS server then points to the authoritative DNS server for the specific domain (e.g., for example.com).
    • The authoritative server holds the actual IP address associated with the domain and returns it to the DNS resolver.
  • IP Address is Returned:

    • IP Address is Returned: The resolver sends the IP address back to the user's browser, which can now communicate with the websiteâ€™s server and display the site.


    Recursive queries in summary

    In summary, recursive queries enable users to enter simple domain names, while the DNS resolver efficiently handles the process of obtaining the IP address by querying multiple DNS servers behind the scenes.
